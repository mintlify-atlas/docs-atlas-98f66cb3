# Go Payment Dashboard

Go Payment Dashboard is an example full-stack web application demonstrating modern best practices for building a payment tracking interface. The backend is a Go REST API using the Chi router, the frontend is React 19 with Tailwind CSS and shadcn/ui, and all data is persisted in a PostgreSQL database via type-safe sqlc-generated queries.

## Architecture overview

The project is structured as a monorepo with a clear separation between the Go backend and the React frontend.

```
go-dashboard/
├── cmd/api/          # Application entry point
├── internal/
│   ├── api/          # HTTP handlers and request/response types
│   ├── application/  # Business logic (services, queries, mutations)
│   ├── database/     # Database connection and health
│   ├── domain/       # Domain entities and repository interfaces
│   ├── generated/    # sqlc-generated type-safe query code
│   ├── infrastructure/db/  # Repository implementations
│   └── server/       # Chi router setup and route registration
├── frontend/         # React 19 + Vite + Tailwind CSS frontend
├── sql/
│   ├── migration/    # PostgreSQL DDL migrations
│   └── query/        # sqlc SQL query definitions
├── compose.yml       # Docker Compose for local development
└── Dockerfile        # Production container image
```

## Tech stack

| Layer | Technology |
|---|---|
| Backend language | Go |
| HTTP router | [Chi v5](https://github.com/go-chi/chi) |
| Database | PostgreSQL |
| Query generation | [sqlc](https://sqlc.dev) |
| Frontend framework | React 19 + Vite |
| UI components | shadcn/ui + Tailwind CSS v4 |
| Data fetching | TanStack Query v5 |
| Charts | Recharts |
| Drag and drop | dnd-kit |
| Local dev | Docker Compose + Air (live reload) |
| Deployment | Fly.io |
| CI/CD | GitHub Actions |

## Key features

- **Payment CRUD** — create payments with sender, recipient, and amount; list all payments; look up a payment by ID
- **Type-safe database layer** — sqlc generates Go code directly from SQL, eliminating hand-written query boilerplate
- **Live-reload development** — Air watches Go source files and restarts the server automatically inside Docker
- **React dashboard UI** — sortable table with drag-and-drop columns, charts via Recharts, and a form for creating payments
- **Automated deployments** — push to `main` triggers tests, a semantic release, and deployment to Fly.io
