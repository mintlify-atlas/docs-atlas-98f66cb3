The Go Payment Dashboard exposes a JSON REST API for managing payments and checking service health. No authentication is required.

## Base URLs

| Environment | URL |
| ----------- | --- |
| Production  | `https://go-dashboard.fly.dev` |
| Local       | `http://localhost:8080` |

## Content type

All requests that include a body must set the `Content-Type` header to `application/json`.

```http
Content-Type: application/json
```

## Authentication

This API does not require authentication. No API keys, tokens, or credentials are needed.

## CORS

The API is configured with the following CORS policy:

| Setting | Value |
| ------- | ----- |
| Allowed origins | Configured via the `FRONTEND_ORIGIN` environment variable |
| Allowed methods | `GET`, `POST`, `PUT`, `DELETE`, `OPTIONS`, `PATCH` |
| Allowed headers | `Accept`, `Authorization`, `Content-Type` |
| Credentials | Allowed |
| Max age | 300 seconds |

## Error format

All error responses return a JSON object with a single `error` field:

```json
{
  "error": "descriptive error message"
}
```

## Endpoints

| Method | Path | Description |
| ------ | ---- | ----------- |
| `GET` | `/payments` | List all payments |
| `POST` | `/payments` | Create a new payment |
| `GET` | `/payments/{id}` | Get a payment by ID |
| `GET` | `/health` | Application health check |
| `GET` | `/health/db` | Database health check |
